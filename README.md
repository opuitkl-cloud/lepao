# 武汉理工校园跑
# WHUT绘跑

> GPS 轨迹绘制 + 武汉理工大学智慧体育自动提交工具

在地图上绘制跑步轨迹，一键提交到 WHUT 智慧体育系统。支持桌面和移动端。

---

## 功能

- **轨迹绘制** — 选择校园地图，用鼠标/触屏绘制跑步路线
- **打卡点管理** — 自动加载校区标准打卡点，点击切换激活/取消
- **一键提交** — 通过智慧体育链接登录 → 验证 → beforeRun → OSS 上传 → stopRun 全自动
- **登录过期检测** — 提交前自动检查 token 有效性，过期自动退回登录页
- **移动端适配** — 双页面布局

## 安全说明
根据链接token是不能反推账号密码的。其实也是因为最开始没有找到登录页面才用的链接，不过也挺好

## 快速开始

### 前提

- Node.js ≥ 18（需要 `fetch` 原生支持）

### 安装 & 运行

```bash
cd huaipao
npm install
node server.js
```

打开浏览器访问 **http://localhost:6660**

### 使用说明

1. **登录** — 打开"智慧体育"→ 右上角 → 浏览器打开 → 复制链接，粘贴到登录页
2. **选择地图** — 点击南湖/鉴湖/西院/余区/东院切换底图
3. **选打卡点** — 点击打卡点激活（粉色=已激活，青色=未激活）
4. **画轨迹** — 点击"开始绘制"在地图上画线，再次点击结束
5. **提交** — 点击⚡确认，自动检查登录 → 生成轨迹 → 提交到 WHUT

### 默认参数

| 参数 | 默认值 |
|------|--------|
| 运动总时间 | 666 秒 |
| 采样间隔 | 6 |
| 跑步模式 | 计分跑 |

## 技术栈

**后端：** 纯 Node.js 内置 `http` 模块（无 Express 依赖）
- AES-128-CBC 加密（crypto-js）
- MD5 参数签名
- OSS 阿里云 STS 上传

**前端：** 纯 HTML/CSS/JS SPA（无框架）
- Canvas 绘图
- GPS ↔ 像素坐标双向转换
- Haversine 距离计算

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/whut/login` | POST | 登录，返回 auth |
| `/api/whut/check-auth` | POST | 检查登录是否过期 |
| `/api/whut/submit` | POST | 创建跑步任务，返回 jobId |
| `/api/whut/job/:id` | GET | 轮询任务状态 |
| `/api/whut/history` | GET | 历史记录 |
| `/api/settings` | GET/POST | 图层配置持久化 |
